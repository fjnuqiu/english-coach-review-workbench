# English Coach 课程复习工作台设计

## 目标

把当前按日期和功能 Tab 堆叠的管理台，改成以“课程”为第一层、以“复习”为唯一主任务的学习工作台。用户打开页面后只需要选择课程并完成当天到期复习，不再在“今日学习 / 复习台 / 学习回传 / 语音播放”之间切换。

## 调研结论

- Anki 的有效结构是“卡组 -> 今日新卡 / 学习中 / 待复习 -> 逐张作答”，并且先显示问题、用户回忆后再显示答案。
- Quizlet 的有效结构是“学习集 -> 个性化学习会话”，允许书写作答、选择复习范围和按掌握度推进。
- Duolingo 的有效结构是“课程 -> 单元”，把复习融进课程路径，并将中级内容拆成更短、更聚焦的小单元。
- 本项目不复制品牌视觉，只采用这三个行为模式：课程分组、当天到期任务、主动回忆后反馈。

## 信息架构

页面只保留一个主界面：

1. 顶部栏
   - 品牌：English Coach
   - 今日进度：待复习、已完成、连续学习天数（有数据时）
   - 工具按钮：导入学习材料、中译英
   - 工具以弹层打开，不占据页面主体，不再作为 Tab
2. 课程列表
   - 每个课程显示：课程名、简短中文摘要、来源日期、总句数、今日到期数、掌握进度
   - 默认只展开最优先课程；其他课程保持紧凑
   - 点击课程标题展开或收起
3. 课程复习区
   - 展开课程后显示课程摘要和“开始 / 继续复习”按钮
   - 进入复习后一次只显示一道题
   - 中文完整句在上，英文参考答案默认隐藏
   - 支持文字输入、开始录音、结束录音、检查答案、播放参考英文
   - 检查后标红漏词和多余词；核心语义达到阈值即可通过
   - 保留“上一个 / 下一个”和“会 / 不熟 / 不会”
4. 课程完成状态
   - 当天该课程到期句完成后显示完成状态和下一次复习时间
   - 没有到期题时可手动“复习全部”，但不作为默认任务

## 视觉方向

- 安静、清楚、偏学习工具，不做营销首页。
- 背景使用浅灰，内容以白色工作区和细边框区分；主色使用墨绿，反馈使用蓝、琥珀和红。
- 课程项最大圆角 8px，不嵌套装饰卡片。
- 桌面宽度控制在约 1120px；手机端单列，底部主要操作按钮保持可触达。
- 标题层级收紧：页面标题、课程标题、题目三级，不使用超大字号。

## 数据模型

新增 `state/courses.json`，课程是复习卡的归属实体。

```json
{
  "id": "pet-food-safety",
  "title": "宠物饮食安全",
  "summary_zh": "学习宠物不能食用的食物和安全提醒。",
  "source_files": ["normal_video.mp4"],
  "learned_on": "2026-07-11",
  "card_ids": ["kittens-cannot-drink-cow-milk"],
  "order": 1
}
```

`review-items.json` 中每张卡新增 `course_id`。旧数据一次性迁移：

- 客户沟通基础
- Cindy 的卢浮宫之旅
- 卢浮宫购票与馆内服务
- 宠物饮食安全
- 宠物用品
- 宠物护理与情感连接
- 幼犬习性
- 养宠决定与建议

上传整理结果从单个 `title / review_cards` 扩展为 `courses[]`。每个课程有自己的标题、摘要、学习点和复习卡；保留旧格式兼容已有任务。

## API 与交互状态

`/api/dashboard` 增加 `courses`：

- 每门课程返回 `cards`、`due_count`、`completed_today`、`total_count`、`mastered_count`
- `review_cards` 仍保留一段时间，供旧前端兼容
- `/api/review-result` 继续按卡片 ID 记录结果，课程进度由卡片实时聚合

页面需要覆盖：加载中、服务断开、没有课程、课程无到期题、录音中、识别失败、答案检查后、课程当天完成。

## 复习规则

- 首次入库只标记为 `pending`，不算“不熟”，也不算今天已完成。
- `不会`：次日复习。
- `不熟`：次日或当前最短间隔复习。
- `会`：按 2、4、7、14、30 天递增。
- 每日只把到期卡排进课程会话，避免一次把整门课程全部压给用户。

## 测试与验收

- 后端：课程聚合、旧数据迁移、多课程上传结果、复习结果只影响对应卡、首次入库不计完成。
- 前端静态契约：旧四个 Tab 文案和容器消失；课程列表、展开按钮、课程进度和复习工作区存在。
- 浏览器流程：展开课程、开始复习、答案默认隐藏、文字检查、语音开始/结束、标记结果、上下题。
- 视口：桌面 1440x900、手机 390x844；无文本溢出、无控件重叠、无横向滚动。

## 明确不做

- 不引入排行榜、徽章或社交功能。
- 不保留独立的语音播放页面。
- 不在页面主体展示 Markdown 原始笔记；原始笔记仍留在本地文件中用于追溯。

## 2026-07-16 网站化补充

- 使用 Supabase Auth 的独立邮箱/密码账号，不复用参考项目的固定共享密码。
- 课程与复习卡按 `user_id` 分表存储，并启用 RLS；本地现有数据仅在用户登录后迁移。
- GitHub Pages 只承载静态前端；本地 Codex 文件整理和中译英不会把密钥或个人上传暴露到浏览器。
- 课程默认按 0-100 熟练度升序排列，并显示熟练度标签和进度条；可切换为待复习数量或原课程顺序。
