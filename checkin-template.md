# Check-in Template

## 标准版

```text
日期：
时间段：早地铁 / 午间 / 晚地铁 / 夜练
A+课程：
完成了：
卡住了：
今天新学：
不熟的内容：
不会的内容：
自评：听力__/3，表达__/3，发音__/3，词块__/3
备注：
```

## 极简版

```text
时间段：
完成：
不熟：
不会：
```

这四行最适合手机上快速发送。我会把它解析成学习日志，并据此安排下一次复习。

写法建议：

- 已有复习项尽量写 id，例如 `current-status`。
- 新薄弱表达可以直接写英文或中文，我会先加入复习队列。
- `不熟` 会按 `shaky` 处理，明天复习。
- `不会` 会按 `fail` 处理，明天复习。

## 一句话版

很忙时可以只发一句。系统会保守识别明确的 `完成`、`不熟`、`不稳`、`不会`、`卡住` 标记：

```text
晚练完成 A+ 第 1 课，30 秒项目进展说明，blocker 不熟，schedule risk 不会
```

这会被解析为：

- 时间段：夜练
- 完成：A+ 第 1 课，30 秒项目进展说明
- 不熟：blocker
- 不会：schedule risk

## 夜练额外建议

晚上可以补充这 3 项，方便我判断你是否能用于客户沟通：

```text
今天开口说了多久：
最卡的一句话：
希望下次重点练：
```

## 自评标准

| 分数 | 含义 |
| ---: | --- |
| 0 | 完全不会，无法回忆 |
| 1 | 见到能懂，但自己说不出来 |
| 2 | 能说出来，但慢、犹豫或不自然 |
| 3 | 能较自然地使用 |

## 本地记录命令

当你把打卡内容发回来后，我会使用：

```bash
python3 english-coach/tools/coach.py checkin --date YYYY-MM-DD --text "<你的打卡内容>"
```

学习日志会追加到 `state/checkins.jsonl`，复习队列会同步更新到 `state/review-items.json`。
